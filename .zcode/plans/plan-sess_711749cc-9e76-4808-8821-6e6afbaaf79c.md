# Migrate Ahnajak Topup from Supabase to MySQL

## Goal
Replace the entire Supabase stack (Postgres + Auth + Edge Functions + Realtime + Storage + RLS) with a self-contained **MySQL + Express API + JWT auth** setup, so cloning the project = set MySQL credentials → run 2 commands → done.

## Clone experience (the end result)
```bash
git clone <repo> && cd <repo>
cp .env.example .env      # fill in MySQL + JWT_SECRET
npm install
npm run db:migrate        # creates all 24 tables + views
npm run db:seed           # inserts default settings + admin user
npm run dev               # starts API server + Vite frontend together
```

## Phase 2 — Database (`database/`)

### `database/schema.sql` — consolidated from all 70 migrations
One file, all 24 tables + 1 view, MySQL-native types:

| Postgres | MySQL |
|---|---|
| `UUID DEFAULT gen_random_uuid()` | `CHAR(36) DEFAULT (UUID())` |
| `TIMESTAMP WITH TIME ZONE DEFAULT now()` | `DATETIME DEFAULT CURRENT_TIMESTAMP` |
| `JSONB` | `JSON` |
| `BOOLEAN` | `TINYINT(1)` |
| `TEXT[]` arrays | `JSON` |
| `app_role` ENUM | `ENUM('admin','user')` |
| `auth.users(id)` FK | `users(id)` (our own table) |
| `ON CONFLICT DO UPDATE` | `INSERT ... ON DUPLICATE KEY UPDATE` |

Tables: `users`, `profiles`, `user_roles`, `site_settings`, `games`, `packages`, `special_packages`, `topup_orders`, `preorder_games`, `preorder_packages`, `preorder_orders`, `events`, `coupons`, `payment_gateways`, `payment_gateways_public` (VIEW), `payment_qr_settings`, `point_exchange_configs`, `point_transactions`, `wallet_transactions`, `api_configurations`, `game_verification_configs`, `g2bulk_products`.

RLS policies + `SECURITY DEFINER` functions → **dropped** (authorization moves to Express API layer).

### `database/seed.sql` — default data
- Default `site_settings` rows (from `SiteContext.tsx` defaults)
- `payment_gateways` rows (ikhode-bakong, khqrcc) with `enabled=0`
- Default admin: `admin@ahnajak.com` / `admin123` (bcrypt hash pre-computed) + `user_roles` admin row

### `scripts/migrate.js` + `scripts/seed.js`
Connect with `mysql2/promise`, execute SQL files. Idempotent.
